## 用户故事

**作为** OneClawShadow 的用户，
**我想要** 创建和管理具有不同 system prompt 和工具配置的 AI Agent，
**以便** 我可以针对不同的任务和使用场景定制 AI 的行为。

### 典型场景
1. 用户首次打开应用，使用内置的"通用助手"Agent 开始对话。
2. 用户创建一个自定义的"写作助手"Agent，配置专注于写作的 system prompt，不启用文件访问工具。
3. 用户在与通用助手的对话中途，意识到需要专业 Agent，切换到他们的"数据分析师"Agent，对话历史保留。
4. 用户克隆内置的通用助手，创建一个修改了 system prompt 的自定义版本。
5. 用户删除一个不再需要的自定义 Agent。

## 功能描述

### 概述
Agent 管理允许用户创建、配置和管理 AI Agent。Agent 是一个预配置的 AI 角色，由名称、描述、system prompt、可用工具集和首选模型/provider 定义。应用内置默认 Agent 作为起点。用户可以克隆内置 Agent 并创建完全自定义的 Agent。Agent 可以在对话中途切换，改变 system prompt 和可用工具，同时保留对话历史。

### Agent 数据模型

每个 Agent 包含：

| 字段 | 必填 | 说明 |
|------|------|------|
| ID | 是 | 唯一标识符，自动生成 |
| 名称 | 是 | 显示名称（例如"通用助手"、"写作助手"） |
| 描述 | 否 | 简短描述该 Agent 的用途 |
| System Prompt | 是 | 发送给模型的系统提示词 |
| 工具集 | 是 | 该 Agent 可用的工具列表（可为空） |
| 模型/Provider | 否 | 该 Agent 的首选模型和 provider。未设置时使用会话或全局默认值 |
| 是否内置 | 是 | 是否为内置 Agent（只读） |
| 创建时间 | 是 | 创建时间戳 |
| 更新时间 | 是 | 最后更新时间戳 |

### 内置 Agent

应用内置以下 Agent：

1. **通用助手**（默认）
   - System prompt：通用的有用助手提示词
   - 工具集：所有可用的内置工具
   - 这是新会话的默认 Agent

未来版本可能添加更多内置 Agent。内置 Agent 作为模板和参考，帮助用户了解如何配置自己的 Agent。

### Agent 管理操作

#### 查看 Agent 列表
- 以列表形式展示所有 Agent（内置优先，然后是按最后更新时间排序的自定义 Agent）
- 每个列表项显示：Agent 名称、描述（截断）、内置标记（如适用）
- 点击 Agent 打开其详情/编辑视图

#### 创建 Agent
- 用户点击"创建 Agent"按钮
- 填写：名称（必填）、描述（可选）、system prompt（必填）
- 从完整工具列表中选择可用工具（复选框）
- 可选择首选模型/provider
- 保存 Agent

#### 编辑 Agent
- 仅自定义（非内置）Agent 可编辑
- 用户可修改所有字段：名称、描述、system prompt、工具集、模型/provider
- 更改保存后，在使用该 Agent 的任何会话中，下一条消息发送时生效

#### 删除 Agent
- 仅自定义（非内置）Agent 可删除
- 删除前需确认对话框
- 如果有现有会话正在使用该 Agent，这些会话回退到通用助手

#### 克隆 Agent
- 所有 Agent（内置和自定义）都可克隆
- 创建包含所有配置的 Agent 副本
- 副本默认命名为"[原名称]（副本）"
- 副本是自定义 Agent，可自由编辑
- 这是用户自定义内置 Agent 的主要方式

### Agent 与会话的关系

- 每个会话有一个"当前 Agent"字段
- 新会话默认使用通用助手
- 用户可以在对话中途切换 Agent
- 切换 Agent 时：
  - System prompt 变更为新 Agent 的 system prompt
  - 可用工具集变更为新 Agent 的工具
  - 所有已有对话历史保留
  - 下一次模型请求使用：[新 system prompt] + [完整对话历史] + [新用户消息]
- 会话不"依赖于"Agent 的存在——如果 Agent 被删除，会话回退到通用助手

### 用户交互流程

#### 创建新 Agent
```
1. 用户导航到 Agent 管理界面
2. 用户点击"创建 Agent"
3. 系统显示 Agent 创建表单
4. 用户填写名称和 system prompt（必填）
5. 用户可选填描述
6. 用户从工具列表中选择可用工具
7. 用户可选择首选模型/provider
8. 用户点击"保存"
9. 系统验证并保存 Agent
10. 用户返回 Agent 列表，新 Agent 可见
```

#### 对话中途切换 Agent
```
1. 用户在活跃的聊天会话中
2. 用户点击聊天工具栏中的 Agent 选择器
3. 系统显示可用 Agent 列表
4. 用户选择另一个 Agent
5. 系统更新该会话的当前 Agent
6. 下一条消息使用新 Agent 的配置
7. 聊天中显示系统指示："已切换到 [Agent 名称]"
```

#### 克隆内置 Agent
```
1. 用户查看内置 Agent 的详情
2. 用户点击"克隆"
3. 系统创建副本，名称后附加"（副本）"
4. 系统以编辑模式打开副本
5. 用户根据需要修改配置
6. 用户点击"保存"
```

## UI/UX 要求

### Agent 列表界面
- 所有 Agent 的列表视图
- 内置 Agent 有视觉区分（例如标记或标签）
- 每项显示：名称、描述摘要、内置标记
- "创建 Agent"按钮（FAB 或顶部操作）
- 点击查看详情，长按或滑动进行快速操作（删除、克隆）

### Agent 详情/编辑界面
- 表单布局，包含以下字段：
  - 名称（文字输入）
  - 描述（文字输入，多行）
  - System Prompt（文字输入，多行，大区域）
  - 工具选择（可用工具列表，带复选框）
  - 模型/Provider（下拉选择器，可选）
- 内置 Agent：所有字段只读，显眼的"克隆"按钮
- 自定义 Agent：所有字段可编辑，"保存"和"删除"按钮

### Agent 切换器（聊天界面中）
- 通过点击顶部应用栏中间的 Agent 名称访问（显示为"[Agent 名称 v]"带下拉箭头）
- 显示底部弹出或弹出菜单的可用 Agent 列表
- 当前 Agent 有勾选标记
- 内置 Agent 有微妙标记
- 点击不同 Agent 立即切换
- 聊天中出现系统消息："已切换到 [Agent 名称]"
- 详见 [UI 设计规范](../../design/ui-design-spec-zh.md) 的详细布局

### 交互反馈
- 保存：确认 toast/snackbar
- 删除：确认对话框，然后成功 toast
- 克隆：打开克隆 Agent 的编辑视图
- 切换：聊天中的内联指示

## 业务规则

### Agent 规则
1. 每个会话必须始终有一个当前 Agent
2. 通用助手是系统默认值，不可移除
3. 内置 Agent 只读；用户必须克隆才能自定义
4. Agent 名称必须非空；允许重名（但 UI 上不鼓励）
5. Agent 的 system prompt 必须非空
6. Agent 的工具集可以为空（该 Agent 只进行聊天，不使用工具）
7. 如果 Agent 未设置模型/provider，则使用全局默认值。模型解析优先级为：
   1. Agent 的首选模型/provider（如已设置）
   2. 全局默认模型/provider（在 FEAT-003 中配置）
   
   这意味着 Agent 的模型设置是可选的——设置时覆盖全局默认值，未设置时优雅回退。这允许用户创建与模型无关的 Agent（仅 system prompt + 工具），也可以创建绑定特定模型的 Agent（例如为了最佳效果始终使用某个特定模型的 Agent）

### 切换规则
1. Agent 切换即时生效；不需要确认对话框
2. 切换不修改过去的消息
3. 切换点记录在会话中（用于展示目的）
4. 如果新 Agent 有不同的工具集，从该点开始只有新工具可用

### 删除规则
1. 删除自定义 Agent 需要用户确认
2. 使用已删除 Agent 的会话自动回退到通用助手
3. 这些会话中的历史消息保持不变
4. 内置 Agent 不可删除

## 非功能性需求

### 性能
- Agent 列表加载时间 < 100ms
- Agent 切换生效时间 < 100ms（不需要网络请求，是本地配置变更）
- 保存 Agent 完成时间 < 200ms

### 数据
- 所有 Agent 数据存储在本地
- Agent 数据包含在 Google Drive 同步中（FEAT-007）
- 内置 Agent 在应用代码中定义，不在数据库中（应用更新时重置）

## 错误处理

### 错误场景

1. **Agent 保存失败（例如存储错误）**
   - 展示：错误 toast"保存 Agent 失败。请重试。"
   - 数据：不做部分保存；要么全部字段保存成功，要么都不保存

2. **会话引用的 Agent 被其他进程删除**
   - 处理：下次加载时会话回退到通用助手
   - 展示：无错误提示；回退是静默的

3. **保存时 system prompt 为空**
   - 展示：表单字段验证错误"System prompt 不能为空"
   - 保存按钮保持禁用状态直到修正

### 功能测试
- 验证首次启动应用时通用助手存在
- 验证内置 Agent 不可编辑或删除
- 验证克隆创建包含所有配置的可编辑副本
- 创建自定义 Agent 并验证所有字段已保存
- 编辑自定义 Agent 并验证更改已持久化
- 删除自定义 Agent 并验证从列表中移除
- 删除正在被会话使用的 Agent，验证回退到通用助手
- 在对话中途切换 Agent，验证 system prompt 变更
- 验证 Agent 切换后对话历史保留
- 验证聊天中显示切换指示
- 验证新会话默认使用通用助手
- 验证 Agent 列表排序（内置优先，自定义在后）

### 边界情况
- 使用最大长度的名称和 system prompt 创建 Agent
- 删除所有自定义 Agent（仅剩内置）
- 在一个会话中快速连续多次切换 Agent
- 创建两个同名 Agent
- 克隆一个 Agent 后删除原始 Agent
