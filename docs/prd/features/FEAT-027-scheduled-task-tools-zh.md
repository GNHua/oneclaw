## 用户故事

**作为** OneClaw 的用户，
**我希望** 能够通过与 AI 代理的对话来管理我的定时任务，
**以便** 我无需切换到定时任务界面，就能列出、运行、更新和删除定时任务。

### 典型场景

1. 用户问"我有哪些定时任务？"代理调用 `list_scheduled_tasks` 并返回一个格式化列表，显示每个任务的 ID、名称、计划、启用状态和最后执行结果。
2. 用户说"现在立即运行我的晨间简报任务。"代理以任务 ID 调用 `run_scheduled_task`。工具通过 WorkManager 将任务加入立即执行队列并返回确认信息。执行完成后用户会收到通知。
3. 用户说"把我的每日简报改为早上 8 点运行，而不是 7 点。"代理以任务 ID 和 `hour=8` 调用 `update_scheduled_task`。只有指定的字段被更改，其他所有字段保持不变。
4. 用户说"删除每周摘要任务。"代理以任务 ID 调用 `delete_scheduled_task`。闹钟被取消，任务被永久移除。
5. 用户说"禁用每日新闻任务。"代理以任务 ID 和 `enabled=false` 调用 `update_scheduled_task`。闹钟被取消，但任务仍保留在数据库中，以便日后重新启用。
6. 用户对一个已禁用的任务说"运行每日简报任务测试一下"。代理调用 `run_scheduled_task`。工具执行该任务，不更改其启用状态或闹钟计划。

## 功能描述

### 概述

FEAT-019 引入了 `schedule_task` 工具，用于通过对话创建定时任务。但任务创建后，用户必须切换到定时任务界面才能管理它们。本功能新增四个内置工具，支持通过对话对定时任务进行全生命周期管理：

| 工具 | 用途 |
|------|---------|
| `list_scheduled_tasks` | 列出所有定时任务及其详情 |
| `run_scheduled_task` | 触发任务立即执行 |
| `update_scheduled_task` | 修改任务配置 |
| `delete_scheduled_task` | 永久移除任务 |

### 工具说明

#### list_scheduled_tasks

返回所有定时任务及其当前状态。无需任何参数。响应包含每个任务的 ID、名称、计划描述、启用状态、最后执行时间和最后执行状态。代理使用这些信息帮助用户通过 ID 识别任务，以便进行后续操作。

#### run_scheduled_task

通过 WorkManager 将定时任务加入立即异步执行队列。工具在加入队列后立即返回确认信息。实际执行在后台使用现有的 `ScheduledTaskWorker` 进行。执行完成后发送通知。关键行为：

- 对已启用和已禁用的任务均有效（适合在不启用计划的情况下进行测试）
- 不重新调度闹钟，也不禁用一次性任务（仅为临时执行）
- 需要网络连接（由 WorkManager 约束条件强制执行）

#### update_scheduled_task

更新现有定时任务的一个或多个字段。使用部分更新语义：只有提供的字段被更改，未提供的字段保留当前值。可更新的字段包括名称、提示、计划类型、小时、分钟、星期几、日期和启用状态。当已启用任务的计划相关字段发生变化时，闹钟会自动重新计算并重新注册。

#### delete_scheduled_task

永久移除定时任务。取消已注册的闹钟并从数据库中删除任务。此操作不可逆。

### 交互模型

用户通过自然语言与这些工具交互。代理根据用户意图判断调用哪个工具。典型工作流程：

1. 用户请求查看任务 -> 代理调用 `list_scheduled_tasks`
2. 代理展示包含任务 ID 的列表
3. 用户通过名称或 ID 引用特定任务 -> 代理调用相应的管理工具

任务通过 ID 进行标识。`list_scheduled_tasks` 工具提供的 ID 供代理用于后续操作。当用户以名称引用任务时，代理也可将任务名称解析为 ID。

## 验收标准

### TEST-027-01：列出所有任务
- **前提条件** 用户已创建一个或多个定时任务
- **当** 代理调用 `list_scheduled_tasks`
- **则** 返回所有任务，包含 id、名称、计划描述、启用状态、最后执行时间和最后执行状态

### TEST-027-02：列表为空
- **前提条件** 不存在任何定时任务
- **当** 代理调用 `list_scheduled_tasks`
- **则** 工具返回一条提示未配置任何任务的消息

### TEST-027-03：立即运行任务
- **前提条件** 存在具有有效 ID 的定时任务
- **当** 代理以任务 ID 调用 `run_scheduled_task`
- **则** 为该任务加入一个 WorkManager 工作请求，工具返回成功消息，确认任务已加入队列

### TEST-027-04：运行任务通知
- **前提条件** 已通过 `run_scheduled_task` 触发某个任务
- **当** WorkManager 执行完成
- **则** 用户收到包含任务名称和响应预览的通知

### TEST-027-05：运行已禁用的任务
- **前提条件** 存在定时任务但已被禁用
- **当** 代理以任务 ID 调用 `run_scheduled_task`
- **则** 任务成功执行，其启用状态和闹钟计划不发生变化

### TEST-027-06：运行不存在的任务
- **前提条件** 不存在具有所提供 ID 的任务
- **当** 代理调用 `run_scheduled_task`
- **则** 工具返回一条提示任务未找到的错误消息

### TEST-027-07：更新任务名称
- **前提条件** 存在定时任务
- **当** 代理以新名称调用 `update_scheduled_task`
- **则** 任务名称被更新，其他所有字段保持不变

### TEST-027-08：更新任务计划
- **前提条件** 存在一个已启用的每日任务，触发时间为 07:00
- **当** 代理以 `hour=8` 调用 `update_scheduled_task`
- **则** 任务的小时更新为 8，闹钟重新计算为下一个 08:00 触发时间，其他所有字段保持不变

### TEST-027-09：更新任务启用状态
- **前提条件** 存在已启用的定时任务
- **当** 代理以 `enabled=false` 调用 `update_scheduled_task`
- **则** 任务被禁用，其闹钟被取消

### TEST-027-10：更新不存在的任务
- **前提条件** 不存在具有所提供 ID 的任务
- **当** 代理调用 `update_scheduled_task`
- **则** 工具返回一条提示任务未找到的错误消息

### TEST-027-11：删除任务
- **前提条件** 存在定时任务
- **当** 代理以任务 ID 调用 `delete_scheduled_task`
- **则** 闹钟被取消，任务从数据库中移除

### TEST-027-12：删除不存在的任务
- **前提条件** 不存在具有所提供 ID 的任务
- **当** 代理调用 `delete_scheduled_task`
- **则** 工具返回一条提示任务未找到的错误消息

## 非功能性需求

- 所有四个工具的超时时间必须为 10 秒（与 `schedule_task` 保持一致）。
- `run_scheduled_task` 在加入队列后立即返回；实际执行超时由 `ScheduledTaskWorker` 控制（10 分钟）。
- 工具注册为 `ToolSourceInfo.BUILTIN`，对所有代理可用。

## 范围外

- 批量操作（例如删除所有任务、禁用所有任务）
- `list_scheduled_tasks` 中的任务筛选或搜索（返回所有任务）
- 通过工具查看执行历史（请使用定时任务界面）
- 创建新任务（已由 FEAT-019 中的 `schedule_task` 覆盖）
