# FEAT-053: 计划任务执行日志

## 文档信息
- **功能 ID**: FEAT-053
- **状态**: 已实现
- **创建日期**: 2026-03-06
- **最后更新**: 2026-03-06

## 背景

RFC-028 引入了计划任务执行功能，详情页展示执行历史列表。每条记录显示时间戳、状态标签和箭头图标——但点击后会无条件跳转到聊天界面，存在以下问题：

1. 聊天界面打开后不加载对应的对话，显示空白。
2. 执行失败的记录没有关联 session，箭头图标被隐藏，行不可点击。
3. 没有任何地方显示**执行失败的原因**。

用户需要一个专门的页面来查看某次执行的详细情况：执行时间、执行结果，以及——最重要的——失败时的错误信息。

## 目标

1. 所有执行历史行均可点击，无论是否存在关联 session。
2. 点击某行后，进入独立的「执行日志」屏幕，展示该次执行的结构化信息。
3. 执行失败时，突出显示错误信息，让用户清楚了解失败原因。
4. 若该次执行产生了对话，提供「查看对话」按钮，点击后跳转至聊天界面并加载对应 session。
5. 修复已有 bug：通过路由参数导航到聊天 session 时，界面不加载对话内容。

## 非目标

- 在执行日志屏幕内嵌入渲染对话消息。
- 在执行日志屏幕提供重试/重新执行功能（已在详情页的「立即运行」中提供）。
- 对执行历史进行筛选或搜索。

## 用户故事

**US-1: 了解失败原因**
作为用户，我希望点击一条失败的执行记录后，立刻看到错误信息，以便知道任务为何未能完成。

**US-2: 查看对话内容**
作为用户，我希望点击「查看对话」按钮后，直接跳转到该次执行所创建的聊天 session，查看完整的对话过程。

**US-3: 查看执行时间**
作为用户，我希望看到执行的开始时间、结束时间和耗时，以便了解任务的执行效率。

## 功能需求

### FR-1: 执行历史行始终可点击
计划任务详情页的执行历史列表中，所有行均可点击，箭头图标始终显示。点击后导航至该记录对应的执行日志屏幕。

### FR-2: 执行日志屏幕
新增全屏目标页，展示以下内容：

- **状态头部**：带颜色的状态标签（运行中 / 执行成功 / 执行失败）和任务名称。
- **时间卡片**：开始时间、结束时间（如有）、计算得出的耗时。
- **错误卡片**（仅失败时显示）：红色卡片，展示 `errorMessage`；若无记录，则显示兜底提示文案。
- **「查看对话」按钮**（仅 `sessionId` 非空时显示）：点击后导航至聊天界面并加载对应 session。

### FR-3: 修复聊天 Session 加载
导航至 `Route.ChatSession` 时，将路由参数 `sessionId` 以 `initialSessionId` 形式传入 `ChatScreen`，通过 `LaunchedEffect` 触发 `ChatViewModel.initialize()`。修复了聊天界面打开后显示空白的已有 bug。

## 非功能需求

- 执行日志屏幕通过新增的 `getRecordById` repository 方法加载数据，该查询为主键查找，性能开销极小。
- 屏幕需优雅处理记录不存在的情况（显示「未找到执行记录」）。
